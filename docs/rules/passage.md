核心数据基础：课文-单词索引

在设计任何阶段之前，需要建立一个关键的数据结构："每个词在课文中出现的原句"的索引。

word: "nature reserve"
→ 原句: "One school in South Africa sits on a nature reserve
with many different types of wildlife."
→ 所在段落: 第2段
→ 段落完整文本: "One school... boarding school..."

这个索引是三个阶段所有功能的共同数据源。它在编写课文数据时手工标注，或通过字符串匹配自动生成。U5L2 有 15 个词，对应课文中 15 处高亮位置。

  ---

目标：学生第一次接触单词时，建立语境记忆而非孤立记忆。

1A. 今日计划页（Today）的入口改造

当日计划卡显示"今日新词 5 个"时，在进入练习前增加一个提示：

┌──────────────────────────────────────────┐
│  Unit 5 · Lesson 2   今日 5 个新词          │
│                                          │
│  📖 建议先读课文，再开始练习                  │
│  这 5 个词全部出现在课文里                    │
│                                          │
│  [先去读课文]      [直接开始练习]              │
└──────────────────────────────────────────┘

点"先去读课文"→ 跳转到阅读页，且今日新词自动高亮显示（橙色）。这是最低成本、最高价值的改动。

1B. 单词卡（Cards）的语境改造

现在：单词卡显示 word / IPA / 释义 / 通用例句（来自数据库 example 字段）

改造后：对有课文索引的词，在卡片上增加一个"课文原句"区块，置于通用例句之上：

┌──────────────────────────────────────────┐
│  nature reserve                           │
│  /ˈneɪtʃər rɪˌzɜːrv/                      │
│  an area of land where animals and        │
│  plants are protected                     │
│                                          │
│  📖 课文原句                               │
│  "One school sits on a [nature reserve]  │
│   with giraffes, snakes and wild boars." │  ← 词高亮
│                                          │
│  通用例句                                   │
│  "We visited a nature reserve..."        │
└──────────────────────────────────────────┘

视觉层级：课文原句在上（暖橙底色框），通用例句在下（灰色）。传达信息："这个词你在课文里见过"。

1C. 闪卡（Flash）的语境模式

在现有闪卡基础上增加一个可切换的"课文模式"开关（页面顶部）：

- 标准模式（现在）：正面 = 词，背面 = 中文翻译
- 课文模式（新增）：正面 = 课文原句（词用 _____ 遮住），背面 = 词 + 释义 + 完整原句

这样学生在语境中猜词，而不是在词中猜意思——认知方向完全反转，效果更强。

阶段二 — 阅读（During）：课文里强化单词

目标：阅读过程本身就是一次有结构的单词复习。

2A. 阅读页基础设计

阅读页布局（移动端优先）：

│ Level 1 学习中 │ 蓝底色 + 下划线   │ 正在记忆中               │
├────────────────┼───────────────────┼──────────────────────────┤
│ Level 2 熟悉   │ 紫色细下划线      │ 已有印象                 │
├────────────────┼───────────────────┼──────────────────────────┤
│ Level 3 掌握   │ 绿色极细下划线    │ 基本掌握，不干扰阅读     │
└────────────────┴───────────────────┴──────────────────────────┘

点击词弹出轻量卡片（底部滑出，非全屏）：词 + IPA + 释义 + 原句高亮。

2B. 段落末尾"遮词回想"小测验（核心功能）

每段读完后，在段落下方弹出一个小测验卡片（不强制，可跳过）：

┌──────────────────────────────────────────┐
│  💭 你还记得吗？                            │
│                                          │
│  "Students ___________ problems facing  │
│   the plants and animals on the         │
│   reserve."                             │
│                                          │
│  [discuss]  [solve]  [imagine]  [report]│  ← 4选1
│                                          │
│  [跳过]                                  │
└──────────────────────────────────────────┘

答对→ 绿色动效 + 该词在段落中由橙变绿 + 掌握度+1（等同于一次正确练习）

答错→ 展开段落，词高亮闪烁，原句重新呈现，掌握度不变

触发逻辑：
- 只针对该段落中掌握度 0–1 的词（未掌握/学习中）
- 每段最多触发 1 次（选该段中掌握度最低的词）
- 学生已向下滚动超过段落高度 80% 才触发（避免打断阅读节奏）

两大记忆原理的自然嵌入：
- 主动提取（Active Recall）：学生凭记忆从上下文猜词，而不是被动认识
- 间隔重复（Spaced Repetition）：答题结果直接更新掌握度数据，与练习模块共用同一套计数

2C. 两种阅读模式（顶部切换）

┌─────────────┬────────────────────────────────────┐
│    模式     │                说明                │
├─────────────┼────────────────────────────────────┤
│ 📖 专注阅读 │ 只显示高亮词，无小测验，轻松阅读   │
├─────────────┼────────────────────────────────────┤
│ 🧠 学习模式 │ 启用段落末尾小测验，掌握度实时更新 │
└─────────────┴────────────────────────────────────┘

默认进入"学习模式"（因为这是学习 app）。若学生只想流畅阅读可切换。

阶段三 — 复习（After）：课文句子出题

目标：练习题的素材直接来自课文，学生会有"这句话我见过"的熟悉感，记忆强化更深。

3A. 练习模块新增"课文填空"题型（Type C）

现有题型：
- Type A：词 → 选义项
- Type B：义项 → 填词

新增 Type C：课文语境填空

┌──────────────────────────────────────────┐
│  Type C · 课文语境                         │
│                                          │
│  "Their _____________ isn't just about  │
│   homework and exams. Students solve    │
│   real world problems..."               │
│                                          │
│  [education]  [friendship]              │
│  [boarding school]  [understanding]     │
│                                          │
│  答对后：                                  │
│  ✅  正确！查看原文上下文 →                   │
└──────────────────────────────────────────┘

"查看原文上下文"：点击后跳转阅读页，自动滚动到该词所在段落，词高亮闪烁 2 秒。这是三个阶段之间最直接的闭环连接。

3B. 沉浸模式（ImmersiveMode）的语境增强

现有沉浸模式已有 vocab / practice 两种。新增第三种：

- 课文语境模式：全屏逐题，每题均为课文原句填空（Type C）
- 进入方式：在阅读页完成全文后，出现"开始语境练习"按钮 → 进入此模式

3C. 单词卡"答题后跳转"

练习模块（非沉浸）答完某词后，题卡底部显示：

[查看课文原句 →]

点击跳转阅读页，该词的段落自动滚动到屏幕中央，高亮闪烁。让学生把抽象的"练习正确"与"真实语境"建立连接。

计划模块的缺口

现状

  ---
计划模块需要的两个改动

改动一：今日计划页（daily）的学习流程

现状：今日计划页直接列出"今日新词"和"巩固词"，点击进入练习。

改造后：当今日词汇来自有课文的 lesson 时，计划页的任务列表变成三步式：

今日任务 · Unit 5 Lesson 2

① 📖 读课文         [未完成]  →  跳转阅读页
② 🆕 新词练习 5个   [未完成]  →  需完成①后解锁（或可跳过）
③ 🔄 巩固词练习 8个  [未完成]  →  随时可做

关键设计决策：
- ①不强制——学生可以直接跳到②，但有视觉提示"先读课文效果更好"
- "已读课文"的判定：进入阅读页并滚动超过 50% 即视为完成，不要求答对小测验
- 读完课文后回到计划页，①标记为完成，②的提示消失

为什么重要：这是让"先读课文"从一个功能变成一个习惯的机制。不是靠学生主动记得，而是计划页主动引导。

这是对记忆效果影响最大的改动，也是方案里完全缺失的部分。

现状：巩固词练习使用通用例句（数据库 example 字段）。

改造后：巩固词练习优先使用课文原句出题。

具体逻辑：

↓ 是
课文索引里有这个词的原句？
↓ 是
用课文原句做填空题（Type C）
↓ 否
回退到通用例句（现有逻辑）

[⭐] 是独立的第三个按钮，语义是"这课是本周重点"：
