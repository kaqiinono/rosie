# 关卡设计框架初步构想
计算模块可以分为六大关卡，循序渐进：
## 第一关：数感与基础认知（基石）
建立对数字的直觉，包括数的大小、估算、数位概念，这是后续所有计算的前提。
## 第二关：加减法
从口算到竖式，从不进位到进退位，再到连加连减和混合运算，难度梯度清晰。
## 第三关：乘除法
乘法表是核心，向上延伸到多位数乘除、余数、因数和倍数。
## 第四关：括号与运算顺序
这是一个容易被忽视的独立关卡——先乘除后加减、括号改变顺序，是很多学生的障碍点。
## 第五关：分数 · 份数 · 小数
三者相互关联，通分约分、小数互转、份数平均分配都在这里融合。
## 第六关：综合运算与应用题
将前五关的能力整合成解决实际问题的能力，包括多步计算和文字题列式。

# 每关的难度分级（星级或子关卡设计）
## 难度分级原则
每关内部分三档——入门、进阶、挑战。数字范围、运算步骤数、抽象程度依次递进，学生先解锁入门再解锁进阶，避免直接暴露在超纲题前。
## 出题逻辑
题目有两个维度动态生成：一是数字范围（由难度档位决定），二是结构变体（如加法可以是"a+b=?"或"a+?=c"或"?+b=c"，同一知识点出现不同形态，防止学生只记答案）。
## 评分系统设计
四层叠加：基础分（答对得分）→ 速度奖励（在时限内完成额外得分）→ 连击加成（连续答对触发倍率）→ 通关满分奖励。这样设计让不同水平的学生都有动力——慢但准的可以靠正确率通关，快而准的可以刷高分。
## 通关门槛
正确率要求从第一关70%逐步提升到第六关85%，既有梯度又留有容错空间，不会因为一道难题直接卡关。

## 第一层：知识点选择策略
### 自适应选择权重

系统维护每个知识点的权重分，权重越高越容易被抽到出题：

```
weight[topic] = baseWeight
+ 0.4 × (1 - masteryRate)   // 掌握率越低，权重越高
+ 0.3 × timeSinceLastSeen   // 久未出现，权重升高
- 0.2 × recentCorrectStreak // 连对越多，权重降低
+ 0.1 × levelPriority       // 当前关卡重点知识加权
```
每次出题从权重分布中按概率采样，而非轮流出题——薄弱点自动高频复现。

### 题型变体矩阵

同一知识点生成多种结构，防止学生死记答案：


| 知识点 | 变体 A | 变体 B | 变体 C |
| :--- | :--- | :--- | :--- |
| 加法 | a + b = ? | a + ? = c | ? + b = c |
| 乘法 | a × b = ? | ? × b = c | 积是c，写出一组因数 |
| 括号 | 填写结果 | 填入括号使等式成立 | 判断步骤是否正确 |
| 分数 | 计算结果 | 化简分数 | 比较大小排序 |


## 第二层：参数生成规则
### 难度参数化
每个参数独立可控，组合决定最终难度：

1. 数字范围
* ⭐ 1–20 / ⭐⭐ 1–999 / ⭐⭐⭐ 1–9999
2. 进位/借位次数
* ⭐ 0次 / ⭐⭐ 1次 / ⭐⭐⭐ 2+次
3. 运算步骤数
* ⭐ 1步 / ⭐⭐ 2步 / ⭐⭐⭐ 3+步
4. 干扰项设计
* ⭐ 随机 / ⭐⭐ 常见错误值 / ⭐⭐⭐ 陷阱选项

### 参数合法性约束
生成随机参数后，必须通过以下验证才能出题，否则重新生成：
```
constraints = {
  除法: divisor != 0, result >= 0, preferIntegerResult,
  分数: denominator in [2,3,4,5,6,8,10,12],
  减法: a >= b,  // 不出现负数（基础关卡）
  应用题: answerIsPositiveInteger,
  难度: estimatedSolveTime within [0.5×, 1.5×] timeLimit
}
```

## 第三层：干扰项生成策略
### 选择题干扰项（4选1）

| 干扰项类型 | 生成规则 | 目的 |
| :--- | :--- | :--- |
| 进位遗漏 | 正确答案 - 进位值 | 检测进位掌握 |
| 顺序混淆 | 交换运算顺序的结果 | 检测括号理解 |
| 位值错误 | 答案数位移一位 | 检测数位概念 |
| 相近数字 | 答案 ±1 或 ±10 | 要求精确计算 |

学生选了哪个干扰项，直接揭示其错误模式，输入追踪系统。

# 薄弱点追踪
## 错误数据采集维度
* 时间维度 答题用时、在哪一步停顿最久、是否在最后时刻修改答案
* 错误维度 选了哪个干扰项、连续错几次、同类题错误率
* 重试维度 是否反复在同一知识点卡壳、错误后纠正速度
* 时序维度 知识点上次出现到现在的遗忘曲线衰减程度

## 遗忘曲线与复习调度
间隔重复算法（SM-2 简化版）

每个知识点维护一个复习间隔，根据上次答题表现动态调整：
```
初次答对  → 下次复习间隔 = 1 天
再次答对  → 间隔 × 2.5（最长 21 天）
答错      → 间隔重置为 1 天，权重提升
连续答对3次 → 标记为「已掌握」，低频维护
```
每次进入关卡，系统优先插入「即将遗忘」的知识点（距复习日期最近的优先）。

## 薄弱点报告结构
每次闯关后生成

| 报告模块 | 内容 | 用途 |
| :--- | :--- | :--- |
| 本次概览 | 正确率、用时、最快/最慢题 | 即时反馈 |
| 错误详情 | 每道错题 + 正确解法步骤 | 当下纠错 |
| 薄弱点列表 | Top3 高频错误标签 | 重点关注 |
| 成长趋势 | 近5次同类题正确率折线 | 感知进步 |
| 下次建议 | 推荐练习的知识点和题型 | 引导下一步 |

切换顶部的两个标签可以分别查看出题算法和薄弱点追踪系统。以下是几个关键设计决策的说明：
1. 为什么用算法生成而不用题库
题库有限，学生刷完一轮会记住答案而不是真正掌握。算法生成可以保证无限不重复，同时每道题的难度参数精确可控——题库很难做到"恰好出一道有两次进位的三位数加法"这种精细控制。
2. 干扰项是最有价值的数据
学生选了哪个错误选项，比"对还是错"包含多十倍的信息。选"进位遗漏型"干扰项，说明理解了加法但进位规则不稳；选"位值错误"干扰项，说明数位概念有问题——这两种错误需要完全不同的补救方式。
3. IRT 能力估计保持 75% 正确率的意义
心理学研究表明，75% 左右的正确率对应"学习区"（既不太难产生挫败感，也不太简单缺乏挑战）。低于 60% 学生会焦虑放弃，高于 90% 则没有成长。自适应难度的核心目标就是把每个学生稳定在这个区间里。
4. 遗忘曲线调度解决的真实问题
很多练习 App 只关注当下对错，不管上周学过的有没有遗忘。SM-2 间隔重复保证知识点在"快忘但还没忘"的时机出现，这个时间点强化记忆效率最高。

# UI 界面原型（答题界面、报告界面）设计
## 两个界面的核心设计决策说明：
### 答题界面
![img_5.png](img_5.png)

计时器用环形进度条而非数字倒计时，视觉压迫感更低，但时间紧迫时会变红提醒。连击指示器用三个小圆点而非醒目的大图标，低调但能让学生一眼感知自己的状态。选项布局 2×2 九宫格适合触屏操作，每个选项都是足够大的点击区域。答对/答错后立刻显示解析，而不是等到闯关结束——趁记忆热时纠错效果最好。
### 报告界面
![img_6.png](img_6.png)
得分放在最顶部、最大，满足"第一眼想知道得了多少分"的本能需求。星级+徽章让高分学生有炫耀感，但低分学生不会看到刺眼的"失败"字样。趋势折线图让学生看到自己在进步，这比单次分数更有激励价值。错题部分每道题都附有自动打的错误标签（如"运算顺序混淆"），而不是单纯显示对错——这直接连接到薄弱点追踪系统。底部"专项练习"按钮点击后会直接触发针对性出题。

整套产品设计现在已经有了完整闭环：关卡结构 → 出题算法 → 答题体验 → 错误追踪 → 报告反馈。

### 竖式计算
进位格独立成一行小格子，和真实纸上计算的习惯完全一致。学生必须先填进位才能填结果，系统分别判分——进位对了但结果错，说明加法本身有问题；进位填错说明进位概念不稳，两种错误触发不同提示。
### 分数可视化
饼图配合滑块，让学生在操作中感受"1/2 比 1/4 大"而不只是记忆规则。答案输入框是分子/分母分开的两格，防止直接输入小数绕过分数理解。
### 等式填空
比选择题难、比填空题有结构。三道题覆盖加法逆运算、除法逆运算、含括号等式，同一屏可以考察多个知识点，效率更高。
### 数轴拖拽
整数和分数各一条，拖拽有磁吸吸附——整数拖到最近的 0.5 刻度，分数拖到最近的 1/4 刻度，避免因触屏精度问题误判。空间感知能力是很多孩子隐性的薄弱点，这种题型能直接检测出来。
### 应用题分步作答
每一步单独输入并单独给分，过程分比结果分更重要。填写提示（灰色 placeholder 文字）给出算式结构，降低语言障碍，让孩子专注于数学逻辑。每步下方有一行解释，减少因题意不清导致的失分。
