# 如何新增数学讲次（操作说明）

> 面向「我要加一讲新课」——需要准备什么、有哪些做法、文字题和图片题分别怎么处理。
>
> 技术细节（代码模板、注册清单）见 [`docs/add-new-lesson.md`](../add-new-lesson.md)；题目文件约定见 [`new-lesson.md`](new-lesson.md)。

---

## 一句话概览

新增一讲 = **先写好题目源文件** `docs/math/lessons/N.md` → **让 AI 或开发者按规范生成代码** → **本地看一眼页面** → **合并上线**。

系统会自动生成：讲次首页、课前测/课堂/课后/拓展、综合题库、错题本、题海入口、每日计划、组卷等（前提是注册步骤不遗漏）。

---

## 两条途径

| 途径 | 适合谁 | 你怎么做 | AI/开发者做什么 |
|------|--------|----------|-----------------|
| **A. Cursor `/add-lesson`（推荐）** | 有题目文稿，想省事 | 按模板写好 `docs/math/lessons/N.md`；有图则按下文放好图片或组件说明 | 读 N.md，生成数据 + 组件 + 路由，并改 7 处入口注册 |
| **B. 手动按技术文档** | 自己改代码 | 同上准备好 N.md | 你按 [`docs/add-new-lesson/`](../add-new-lesson/) 逐步创建文件 |

**推荐流程：**

1. 复制 [`new-lesson-template.md`](new-lesson-template.md) → 保存为 `docs/math/lessons/N.md`（N = 讲次号，如 `48`）
2. 填入全部题目（见下文「题目文件怎么写」）
3. 在 Cursor 对话里输入：`/add-lesson 48`（或说明「按 add-lesson skill 录入第 48 讲」）
4. 本地 `pnpm dev`，打开 `/math/ny/48` 逐块检查
5. 确认 `/math` 首页、题海、每日计划里能看到新讲次

## 你需要准备什么

### 必交：题目 Markdown 文件

| 文件 | 说明 |
|------|------|
| `docs/math/lessons/N.md` | **唯一必需的「源稿」**；结构见 [`new-lesson-template.md`](new-lesson-template.md) |

文件里应包含（有则写，无则删章节或留空）：

| 章节 | 对应 App 模块 | 无题时 |
|------|---------------|--------|
| `## summary` | 讲次首页文案（主题、口诀、题型介绍） | 可从题目归纳 |
| `## 课前测` | 课前测 | 该模块为空 |
| `## 课堂讲解` | 课堂讲解（可含 `### 模块`、例题、练一练） | 该模块为空 |
| `## 课后巩固` | 课后巩固 | 该模块为空 |
| `## 拓展练习` | 拓展练习 | 该模块为空 |
| `## supplement` / `## 附加题` | 附加题（可选） | 不建 supplement 页面 |

**录入前请自己数一遍题**：例如「第 48 讲共 32 题：课前测 5 + 课堂 12 + 课后 8 + 拓展 7」。漏题是最常见的问题。

### 每道题建议写清的内容

对**每一道**题，在 N.md 里尽量写全（AI 会据此生成数据）：

- 题号与标题（如「例题 3 · 排队问题」）
- 题目正文（关键数字可用 `**加粗**`）
- 题型归属（便于分 `type1` / `type2` …）
- 解析步骤（多条，对应「题型分析」）
- **最终问什么、单位、标准答案**（数字题必填）
- 难度感受（1–5 星，可让 AI 评）

### 可选：图片与交互组件

见下一节「题目类型与要提供的材料」。

### 不需要你准备的

- 不用自己写 React 页面（除非选手动途径 B）
- 不用改 Supabase 表结构（做题进度等沿用现有表）
- 不用改 `constant.ts`

## 题目类型与要提供的材料

先判断你的题属于哪一类，再决定要不要额外交文件。

```
                    ┌─ 纯文字 + 数字答案 ──────────────► 只写 N.md 即可
                    │
题目 ──┼─ 题解里要配图 ─────────────────► N.md + PNG 放到 public/img/math/
                    │
                    ├─ 题面里要配图（示意图、几何图）──► N.md + 附图或 SVG 需求说明*
                    │
                    └─ 可点击的方格/数独等交互题 ────► N.md 内嵌组件代码块** 或附 tsx 文件说明

* 题面配图最终会做成 SVG 组件（figureNode），你可交扫描图/手绘图作参考
** 若讲义里已有 `<XxxGrid .../>` 形式的组件调用，照抄进 N.md
```

### 1. 纯文字题（最常见）

**提供：** 只需 `N.md` 里的文字。

**特征：**

- 题干是文字叙述（可含 HTML 加粗）
- 孩子在输入框填**数字答案**
- 不需要题面插图，不需要题解插图

**N.md 示例：**

```markdown
### 例题 1 · 排队

小明前面有 3 人，后面有 5 人，这一队一共多少人？

- 解析：前面 3 人 + 小明 1 人 + 后面 5 人
- 解析：3 + 1 + 5 = 9
- 问：一共多少人？
- 单位：人
- 答案：9
- 难度：2
```

生成后：数据文件一般为 `lessonN-data.ts`，答题区是数字输入框。

---

### 2. 题解配图（analysisImg）——「解析里放一张图」

**提供：**

1. `N.md` 中注明哪道题需要题解图
2. 图片文件：`public/img/math/{讲次}-{题号}.png`（或 `.jpg`）

**特征：**

- 图出现在**点开「题型分析」之后**，用来讲推导过程
- 静态图即可（拍照、扫描、导出 PNG 都行）
- **不需要**把图写进题目正文区域

**N.md 写法示例：**

```markdown
### 课前测 5 · 圆圈站队

（题目文字……）

- 解析：……
- 题解配图：41-P5.png
```

**你要做的：** 把 `41-P5.png` 放到 `public/img/math/41-P5.png`。

| 字段 | 显示位置 | 你要交什么 |
|------|----------|------------|
| `analysisImg` | 题解折叠面板内 | PNG/JPG 文件 + N.md 里写文件名 |

---

### 3. 题面配图（figureNode）——「题目下方有一张图」

**提供：**

1. `N.md` 中写清题目，并说明「需要题面配图」
2. **参考图**（照片 / 手绘扫描 / PDF 截图）——给 AI 画 SVG 用  
   或：若你已有 SVG/组件代码，可直接贴在 N.md 里

**特征：**

- 图出现在**题目文字下方**（和题干一起可见）
- 一般是示意图、几何图、一笔画图形等
- 技术实现是 React SVG 组件，不是单纯 `<img>`

**N.md 写法示例：**

```markdown
### 课 2 · 一笔画

下面图形能否一笔画完成？（需要题面配图，参考 attachments/lesson38-fig2.png）

- 解析：……
- 答案：能 / 不能（按实际）
```

| 字段 | 显示位置 | 你要交什么 |
|------|----------|------------|
| `figureNode` | 题面下方 | 参考图或 SVG 需求；生成 `.tsx` 数据文件 |

---

### 4. 交互组件题（方格谜题、数独变型等）

**提供：**

1. `N.md` 中每道题附带**组件调用代码块**（与讲义一致），例如：

```tsx
<ShulianGrid rows={[5]} cells={[[1,1,3], [1,3,2]]} />
```

2. 若讲义有专用组件源码，说明组件名与 props（**不要改 props 名**）
3. 方格类题可复制已有基础设施：见 [`figures.md`](../add-new-lesson/figures.md)（子目录名按题型自定，不必叫 `gong`）

**特征：**

- 孩子在网页上**点选、填格、连线**，不是填数字
- 组件自带「检查」按钮
- 数据文件必须是 `lessonN-data.tsx`

| 对比 | 题解配图 | 题面 SVG | 交互组件 |
|------|----------|----------|----------|
| 你要交 | PNG 文件 | 参考图 / SVG 说明 | N.md 里的 tsx 代码块 |
| 显示在 | 解析里 | 题面下 | 题面下（可点） |
| 数据文件 | `.ts` 可 | 必须 `.tsx` | 必须 `.tsx` |

---

### 5. 倍比图 / 拆解图题（结构化图表，非图片文件）

**提供：** `N.md` 中写清题目，并列出「已知行 / 目标行 / 运算步骤」等结构化信息（或照讲义表格录入）。

**特征：**

- 不是上传 PNG，而是数据里的 `rows`、`rcols`、`ops` 等字段
- 右侧/左侧自动生成倍比图、拆解图
- 常见于归一、倍比类讲次

若你不熟悉字段名，在 N.md 用自然语言写清数量关系即可，交给 `/add-lesson` 填字段。

## 推荐操作步骤（途径 A）

### 第 1 步：建题目文件

```bash
# 在仓库中
cp docs/math/new-lesson-template.md docs/math/lessons/48.md
```

编辑 `48.md`：填标题、`summary`、各章节**全部**题目。

### 第 2 步：准备图片（如有）

```bash
# 题解配图示例
public/img/math/48-P3.png
public/img/math/48-L2.png
```

在 `new-lesson.md` 的讲次表里追加一行（可选，方便自己追踪）。

### 第 3 步：触发录入

在 Cursor 中：

```
/add-lesson 48
```

或自然语言：「按 add-lesson 规范录入第 48 讲，题目在 docs/math/lessons/48.md」。

### 第 4 步：验收

```bash
pnpm dev
```

浏览器检查：

| 检查项 | 地址 |
|--------|------|
| 数学首页有新讲次卡片 | `/math` |
| 讲次首页模块与题数 | `/math/ny/48` |
| 各模块列表与详情 | `/math/ny/48/lesson`、`…/homework` 等 |
| 综合题库筛选 | `/math/ny/48/alltest` |
| 题海能搜到该讲 | `/math/sea` |
| 有图题：题面/题解图正常显示 | 点开对应 `[id]` 页 |
| 交互题：能点选、能检查 | 同上 |

构建检查（合并前）：

```bash
pnpm typecheck
pnpm build
```

## 常见问题

**Q：我只有 PDF/Word 讲义，没有 N.md？**  
先把题目**按章节抄进** `docs/math/lessons/N.md`（或让 AI 根据 PDF 起草一版，你再校对题号和答案）。没有 N.md 就无法走标准流程。

**Q：可以只加几道题、不要完整四模块吗？**  
可以。没有的章节留空，生成时对应数组为 `[]`，侧边栏可不显示该模块。

**Q：文字题和配图题能混在同一讲吗？**  
可以。同一讲里有的题纯文字，有的题 `analysisImg`，有的题 `figureNode`，按题分别标注即可。

**Q：图片必须什么格式？**  
题解配图：`public/img/math/` 下 PNG/JPG，路径写在数据里。题面 SVG 由代码生成，你提供参考图即可。

**Q：录入后入口页看不到新讲次？**  
多半是 7 处「入口注册」漏了一项。用 `/add-lesson` 一般会一并处理；若手改代码，对照 [`registration.md`](../add-new-lesson/registration.md) 逐项勾选。
