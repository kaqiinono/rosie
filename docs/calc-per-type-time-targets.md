# 口算 — 按题型的建议耗时（秒/题）

> **用途：** 家长/产品参考，用于判断「练到什么程度算熟练」、以及未来按题型配置限时/报告时的目标值。
>
> **范围：** Rosie `/calc` 模块全部 41 种单运算积木 + 7 种混合运算骨架 + 竖式/逆运算等答题形式补充说明。
>
> **测量方式：** 从「看到题目」到「按对 ✓」的总耗时，含读题、心算、点键盘（PWA 手机端）。

---

## 1. 两种「时间」不要混用

| 概念 | 说明 | 当前系统 |
|------|------|----------|
| **单题答题时限** | 限时内答对才 +1 熟练度 | `calc-time-limits.ts` 按 bucket 默认 2–8 秒（毫秒） |
| **整组练习时限** | 一次 session 的总倒计时 | 首页/设置：不限 / 1 / 3 / 5 / 10 分钟 |
| **本文建议耗时** | 判断「熟练程度」的目标秒数 | 参考值，尚未写入代码 |

系统默认单题时限 ≈ **「能加熟练度的进阶线」**，不是「非常熟练」线。若目标是熟练度 4–5/5，单题时限宜对齐本文 **高级** 列；入门阶段可放宽到 **进阶** 甚至 **入门**。

---

## 2. 四档定义

| 档位 | 含义 | 判断标准 |
|------|------|----------|
| **入门** | 会算 | 能独立做对，允许心算或竖式思路，偶尔错 |
| **进阶** | 稳定 | 连续 10 题对 8 题以上，不用草稿 |
| **高级** ⭐ | 流利 | 几乎不假思索，错率很低，节奏均匀 — **日常「非常熟练」推荐目标** |
| **超高级** | 自动化 | 像背乘法表一样秒出，适合比赛/抽查，不必每个题型都逼到此档 |

**稳定性比偶发快题更重要：** 连续 20 题里 ≥18 题落在目标档时间内做对，比单题偶尔 2 秒更有参考价值。

手机 PWA 比纸笔口算多 **1–2 秒** 输入成本，评估时留余量。

## 7. 答题形式补充（非独立题型）

同一 block 可能因答题界面不同而需加时。

| 形式 | 适用积木 / 条件 | 相对同级整数心算 |
|------|-----------------|------------------|
| **整数 · 数字键盘** | 默认 | 基准 |
| **逆运算 · 挖空** | 设置开启「包含逆运算」，约 30% 整数题 | +2–4 秒 |
| **小数 · 带小数点键盘** | `dec:*` | 见 §4 |
| **有余数 · 商/余两格** | `div:rem` | 见 §3.4 |
| **分数 · 饼图** | 同分母加减，答案 ≤ 1 | 比分数键盘快 2–4 秒 |
| **分数 · 分数键盘** | 异分母 / 分数乘除 / 答案 > 1 | 见 §5 |
| **竖式 · 加/减** | `add:100a` 及以上、`sub:100a` 及以上 | **+8–15 秒** |
| **竖式 · 乘法** | `mul:2d1d`、`mul:2d` | **+8–12 秒** |
| **竖式 · 除法** | `div:multi` | **+8–12 秒** |

**竖式示例：** `add:1000` 心算高级 8–12 秒 → 竖式高级约 **20–30 秒** 算流利。

## 9. 实操建议

1. **不要所有题型统一一个秒数** — 10 以内加法与「分数÷分数」差一个数量级。
2. **「非常熟练」用高级档 + 稳定性** — 见 §2。
3. **熟练度 5/5 可对应高级** — 超高级留给专项（如每天 5 分钟只练 ×6–9）。
4. **竖式、分数、混合单独一档** — 不与 `7×8` 共用同一套秒数。
5. **先会再快** — 正确率稳定后再压时间，避免为赶时限养成猜答案。

### 一句话总结

- 日常「练到很熟」：**多数整数题瞄准 3–8 秒（高级）**；混合 / 分数 / 竖式 **10–20 秒（高级）**。
- 四档用法：**入门能会 → 进阶能稳 → 高级很熟（推荐目标）→ 超高级竞赛向**。
