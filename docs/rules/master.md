## 一、核心概念与术语

| 术语 | 定义 |
|------|------|
| **session** | 一次完整的练习，建议 25–30 题，结束后统一结算 |
| **单题熟练度** | 每道算式独立维护的 0–5 分值，动态升降 |
| **短期掌握** | 熟练度 = 5，且最近 3 次均在时限内答对 |
| **长期掌握** | 短期掌握后，通过 3 轮间隔复习检验 |
| **冷门题** | 距上次出现超过阈值 session 数的题目 |
| **热身期** | 进入新关卡的前 10 题，不计入升级判定 |
| **答题时限** | 各类型题目的最长允许答题时间，见第十一节 |

## 二、题目数据结构

每道题维护以下字段，所有规则基于此结构运行：

```json
{
  "id": "lv6_6x7",
  "question": "6 × 7",
  "answer": 42,
  "level": 6,

  "proficiency": 3,
  "attempt_count": 7,
  "recent_results": [
    { "correct": true,  "time_ms": 980  },
    { "correct": true,  "time_ms": 1200 },
    { "correct": false, "time_ms": 2800 }
  ],

  "status": "active",
  "short_mastered_at": null,
  "long_mastered": false,
  "review_schedule": [],

  "last_seen_session": 12,
  "queue_position": 3,
  "times_seen_this_round": 0,
  "forced_next": false,
  "consecutive_wrong": 0
}
```

### status 枚举

| 值 | 含义 |
|----|------|
| `active` | 日常练习中，按权重出题 |
| `review` | 短期掌握，进入间隔复习池 |
| `mastered` | 长期掌握，退出日常练习 |
| `forced` | 被标记为"次日优先"，下次 session 最先出现 |

## 三、熟练度规则

### 3.1 分值变化

```
答题结果                         熟练度变化
─────────────────────────────────────────
在时限内答对                     +1
超时答对（用时 > 时限）          +0（不加不减）
答错（无论速度）                 -2
─────────────────────────────────────────
熟练度下限 = 0，上限 = 5
```

### 3.2 短期掌握判定

同一道题同时满足以下全部条件，标记为短期掌握（status → `review`）：

```
条件 1：proficiency = 5
条件 2：recent_results 最近 3 次均为 correct: true 且 time_ms ≤ 时限
条件 3：attempt_count ≥ 3
```

### 3.3 间隔复习与长期掌握

短期掌握后，按以下计划抽查：

| 轮次 | 距短期掌握时间 | 通过要求 |
|------|--------------|----------|
| 第 1 轮 | 第 2 自然日 | 正确率 ≥ 90%，时限内 |
| 第 2 轮 | 第 7 自然日 | 正确率 ≥ 90%，时限内 |
| 第 3 轮 | 第 30 自然日 | 正确率 ≥ 85%，时限内 |

三轮全部通过 → `long_mastered = true`，status → `mastered`，退出日常练习。

**间隔检验失败处理：**
```
任意一轮未通过：
  → proficiency 重置为 3
  → status 重置为 active
  → review_schedule 清空，重新从第 1 轮计时
```

## 四、出题引擎：每次 session 如何选题

### 4.1 session 题目构成（共 25–30 题）

```
优先级  类型                          占比    说明
──────────────────────────────────────────────────────────
P0     forced 题目                   先出    连续答错题，强制最先出现
P1     冷门题（超出阈值未出现）        先出    兜底保障，防止被遗漏
P2     当前关卡 active 题（弱题优先）  ~55%   按权重队列出题
P3     间隔复习到期题（review）        ~15%   当日到期才出现
P4     旧关卡混练题                   ~20%   防遗忘
P5     已掌握题随机抽查               ~10%   低频保温
```

> 若 P0/P1 题数超过上限，则压缩 P4/P5 比例，优先保障核心练习。

### 4.2 各类型出题权重

`active` 题目根据熟练度分配权重，控制出现频率：

```
proficiency 0–1   权重 5（高频，每轮必出）
proficiency 2–3   权重 3（正常频率）
proficiency 4     权重 1（低频，即将掌握）
proficiency 5     权重 0（进入 review 池，退出 active）
```

### 4.3 题目形式变换

同一道题在不同出现轮次中，按以下形式循环变换，防止机械记忆：

```
第 1 次出现：标准形式         6 × 7 = ?
第 2 次出现：逆运算           42 ÷ 6 = ?
第 3 次出现：填空形式         6 × □ = 42
第 4 次出现：标准形式（重复）  6 × 7 = ?
……以此类推
```

> 形式变换仅影响呈现方式，熟练度记录在同一道题上，不拆分。
> 加减法类题目：标准 → 逆运算 → 填空，三种形式循环。

## 五、覆盖率保障：确保所有题都出现

### 5.1 循环队列机制

每个关卡维护一个**循环队列**，保证所有题目在进入下一轮前均出现一次：

```
第 1 轮（强制全覆盖）：
  关卡所有题目按初始顺序依次出现，每题出现一次
  times_seen_this_round 达到 1 → 本轮完成
  本轮完成后重置队列，开始第 2 轮

第 2 轮起：
  在队列顺序基础上叠加权重插队：
    proficiency 0–1 的题：每出现 3 题即插队一次
    proficiency 2–3 的题：按队列正常顺序
    proficiency 4–5 的题：排到队列末尾
```

### 5.2 冷门题强制出现（兜底机制）

额外检查每道题的 `last_seen_session`，超过以下阈值则在下一个 session 强制插入（P1 优先级）：

```
proficiency 0–2   超过 2 个 session 未出现 → 强制插入
proficiency 3–4   超过 4 个 session 未出现 → 强制插入
proficiency 5     超过 8 个 session 未出现 → 强制插入（仅 review 状态题）
mastered 题       超过 30 个 session 未出现 → 随机抽查一次
```

### 5.3 session 结束后更新队列

```
每道题答完后：
  last_seen_session = 当前 session 编号
  times_seen_this_round += 1
  queue_position 按新权重重新排序
```

## 六、反复练习不熟悉题目的策略

### 6.1 按错误类型区分处理

不同原因的答错，采取不同对策，而非统一重复：

```
类型 A：超时但答对（time_ms > 时限，correct = true）
  → 知道答案但未自动化
  → 对策：维持正常出现频率，重点练速度；不降难度

类型 B：答错但方法正确（计算失误）
  → 粗心型错误（consecutive_wrong = 1）
  → 对策：下一轮正常出现，不插队，不降难度

类型 C：连续答错 ≥ 2 次（consecutive_wrong ≥ 2）
  → 真正未掌握
  → 对策：
      1. forced_next = true，下次 session 最先出现
      2. 同时插入变体题（逆运算或填空形式）帮助重建理解
      3. 若连续答错 ≥ 4 次，建议提示家长介入
```

### 6.2 交错练习（避免集中重复）

连续错题不立即重出，而是间隔 3–5 题后再次出现：

```
答错一道题后：
  将该题插入当前 session 队列的"当前位置 + 4"处
  而不是"当前位置 + 1"（立即重出）

原因：强迫大脑重新提取，而非靠短期记忆作答
```

### 6.3 不熟关卡的 session 结构

当关卡整体正确率 < 75% 时，自动切换为"攻坚模式"：

```
攻坚模式 session 结构（共 25 题）：
  5 题   当日最弱题（proficiency 最低的 5 道）
  8 题   本关卡 active 题（权重出题）
  4 题   题目形式变体（逆运算 / 填空）
  4 题   本关卡已掌握题（保温）
  4 题   上一关卡混练（不退步）
```

## 七、过滤已熟练题目

### 7.1 退出日常练习的条件

```
status = mastered（长期掌握）的题目：
  → 完全退出 active 队列
  → 仅在 P5 随机抽查池中保留（每 session 约 2–3 题）
  → 若抽查答对：不做任何处理
  → 若抽查答错：proficiency 重置为 3，status 改回 active
```

### 7.2 降频而非消失

短期掌握（status = `review`）的题目不完全消失，按间隔计划保温：

```
review 题目出现规则：
  仅在对应间隔检验日出现（第 2 / 7 / 30 自然日）
  检验日之外不出现（除非被冷门题机制触发）
  检验时集中出现该关卡所有 review 题，一次性检验
```

### 7.3 已掌握题的混练比例

```
当前关卡状态              已掌握题在 session 中的占比
─────────────────────────────────────────────────
正在练习（practicing）    10%（随机抽查）
abc_passed 后              8%
间隔检验期间               5%
升级后（新关卡练习中）     旧关卡题保留 20% 混练，逐步降至 5%
```

## 八、关卡自动升级规则

### 8.1 升级触发时机

```
每次 session 结束后触发一次升级检查（不在答题中途检查）。
热身期（前 10 题）内不触发。
```

### 8.2 升级三道条件（A / B / C 需同时满足）

**条件 A：覆盖率 + 正确率**
```
✓ 本关卡每道题累计出现次数 ≥ 2
✓ 本 session 整体正确率 ≥ 90%
```
未通过 → 继续本关练习，下次 session 重新检查。

**条件 B：短期掌握占比**
```
✓ 本关卡 short_mastered 题目占比 ≥ 80%
  （达标数见第十节速查表）
```
未通过 → 熟练度低的题权重 × 2，加速推进。

**条件 C：无连续失误**
```
✓ 本关卡内没有任何一道题 consecutive_wrong ≥ 2
```
未通过 → 触发该题 forced_next = true，下次 session 优先出现。

### 8.3 间隔检验（A/B/C 通过后触发）

三个条件同时通过，关卡状态进入 `abc_passed`，触发间隔检验：

| 轮次 | 距 abc_passed 日期 | 通过要求 |
|------|--------------------|----------|
| 第 1 轮 | 第 2 自然日 | 正确率 ≥ 90%，时限内 |
| 第 2 轮 | 第 7 自然日 | 正确率 ≥ 90%，时限内 |
| 第 3 轮 | 第 30 自然日 | 正确率 ≥ 85%，时限内 |

**间隔检验失败：**
```
→ 关卡熟练度整体重置为 3（不归零，保留部分进度）
→ 状态回到 practicing
→ 间隔计时清零，重新从第 1 轮开始
```

**三轮全部通过 → 正式升级**，解锁下一关卡。

### 8.4 关卡状态机

```
practicing
  ↓ A/B/C 同时通过
abc_passed
  ↓ 第 2 天检验通过
review_r1
  ↓ 第 7 天检验通过
review_r2
  ↓ 第 30 天检验通过
mastered → 解锁下一关卡
  ↑
  └─ 任意轮检验失败 → 回到 practicing（熟练度重置为 3）
```

## 九、升级后衔接与降级保护

### 9.1 新关卡热身期

```
进入新关卡后，前 10 题为热身期：
  - proficiency 正常记录
  - 不触发升级条件检查
  - 不计入条件 A 的正确率统计
  - 热身期题目全部来自新关卡（不混旧关卡）

第 11 题起正式开始升级判定。
```

### 9.2 旧关卡保留混练

```
升级后 session 构成：
  55%   新关卡 active 题
  20%   上一关卡混练（防遗忘）
  10%   更早关卡随机抽查
  10%   间隔复习到期题
  5%    已掌握题抽查

新关卡进入 abc_passed 后，旧关卡混练降至 10%。
新关卡 mastered 后，旧关卡混练降至 5%。
```

### 9.3 降级保护

```
降级触发条件（热身期结束后）：
  连续 3 次 session 正确率 < 60%
  → 暂停新关卡
  → 返回上一关卡，该关卡 proficiency 整体重置为 3
  → 重新完成 A/B/C 条件和间隔检验
  → 记录降级事件（时间戳 + 原因），供家长查看
```

### 9.4 升降级提示时机

```
升级提示：session 全部结束后的总结页展示，不在答题中途弹出。
降级提示：同上，同时附上"哪些题需要加强"的简要说明。
```

## 十二、实现注意事项

```
1. 数据粒度
   熟练度、队列位置、冷门计数均为"单题级别"，
   不能用关卡整体均值代替，否则会掩盖个别薄弱题。

2. 时间计算
   所有"自然日"间隔以日期差计算（非小时数）。
   abc_passed_date、short_mastered_at 存储为 ISO 日期字符串（YYYY-MM-DD）。
   用户每次打开 app 时对比当前日期，判断是否到达检验日。

3. 答题计时
   从题目完整显示后开始计时，到用户提交答案时结束。
   网络延迟不计入答题时间。

4. 连续答错判定
   consecutive_wrong 基于该题"历史上最近两次出现的结果"，
   跨 session 持久化，不因 session 结束而重置。

5. 升级检查时机
   仅在 session 全部结束后触发，不在答题中途检查。
   热身期（前 10 题）内跳过升级检查。

6. 形式变换记录
   每道题用 appearance_count 记录累计出现次数，
   appearance_count % 3 决定本次使用哪种形式（0=标准，1=逆运算，2=填空）。

7. 攻坚模式判断
   session 开始前检查上一次 session 的整体正确率，
   若 < 75% 则本次切换为攻坚模式 session 结构。

8. 日志记录
   以下事件写入日志（时间戳 + 关卡 + 原因）：
   - 关卡升级
   - 关卡降级
   - 间隔检验通过 / 失败
   - 题目进入 forced 状态
   供家长报告和调试使用。

9. 关卡状态持久化
   每个关卡维护独立状态对象，包含：
   {
     level_id, status, abc_passed_date,
     review_r1_date, review_r2_date, review_r3_date,
     session_count, warmup_complete,
     last_session_accuracy, consecutive_poor_sessions
   }
```
