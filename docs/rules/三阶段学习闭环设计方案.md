# 三阶段学习闭环——完整设计方案

> 本文档与代码实际状态保持同步。最后更新：2026-05-31，全部 8 项 polish 已落地。

---

## 核心设计原则

**孤立的闪卡记忆 = 单词与语境割裂。** 最有效的记忆发生在"有意义的语境"里。本方案利用课文素材，把现有模块（单词卡 / 计划 / 阅读 / 练习）串联成一个有机的三阶段闭环：

- 阶段一 **预习（Before）**：单词第一次出现时就在课文原句里
- 阶段二 **阅读（During）**：阅读时按掌握度高亮，段落末"遮词回想"即时强化
- 阶段三 **复习（After）**：练习题用课文原句挖空，答完可跳回原文

三阶段共用同一份 `masteryMap`，**从任何入口学习都在推进同一条进度**。

---

## 关键架构决策：⭐ 不闸门功能

`WeeklyPlan.focusLessonKey`（⭐ 重点）**仅是计划模块的标签**，不闸门任何其他模块的功能：

- 计划 UI 上显示金色 ⭐ 角标
- 在 Today 页/阅读页 redirect 中作为**软优先级**（多课文候选时优先 ⭐ 课）
- **其他模块全部按"该课是否有课文"自动判断**，与 ⭐ 完全解耦

```
任意词 entry
  ↓ findPassage(entry.unit, entry.lesson)
  ├─ 有课文 → 单词卡/闪卡显示原句、Practice 启用 Type D、Today 推荐先读
  └─ 无课文 → 走通用例句、不出 D 题、Today 不推荐
```

**含义**：标了 ⭐ 但没课文 → 功能不出现；没标 ⭐ 但有课文 → 功能正常出现。这避免了"标了 ⭐ 才能用"的认知负担。

## 核心数据基础：课文-单词索引

所有阶段功能共用 `src/utils/reading-data.ts` 的数据结构：

```typescript
type ReadingPassage = {
  key: string        // 'u5l2'
  unit: string       // 'Unit 5'
  lesson: string     // 'Lesson 2'
  title: string
  paragraphs: string[]
}

// 工具函数（同文件导出）
findPassage(unit, lesson): ReadingPassage | undefined
findPassageByKey(key): ReadingPassage | undefined
findSentenceForWord(passage, word): { sentence; paragraphIndex } | null
buildWordMatchRegex(words): RegExp        // 长短语优先 + 复数容错
resolveMatchedWord(matched, candidates)   // 匹配回原词条
```

**正则规则**：`\b(word1|word2|...)s?\b` — 多词短语按长度倒序，复数容错（`exam` 匹配 `exams`）。

添加新课文的标准流程见 `~/.claude/commands/add-passage.md`（slash command `/add-passage`）。

## 阶段一 — 预习（Before）

### 1A. Today 页三步式任务

任务卡片按 ①②③ 编号呈现，每一步独立可点击：

```
① 📖 读课文          [未完成] → 阅读页（任意有课文的今日新词所属课）
② 🆕 必记词练习 N个   [未完成] → 周计划页（必记子任务）
                              💡 建议先读课文（仅 ① 未完成时显示）
③ 🔖 预习词练习 N个   [未完成] → 周计划页（预习子任务，仅有预习词时显示）
```

- ① 不强制，学生可直接跳到 ②
- "已读课文"判定：进入阅读页且 scroll 超过 50%（`WeekDayProgress.readingDone`）
- 三步式按需出现：没有必记词不显示 ②；没有预习词不显示 ③
- 完成的步骤用绿色 + ✓ 显示，未完成步骤显示橙/蓝/紫色阶任务图标

实现：`src/components/today/TodayDashboard.tsx#ThreeStepRow`。

### 1B. 单词卡课文原句区块（共存例句）

任何有课文的词，单词卡正面同时显示：

- **📖 课文原句**（暖橙底色框）：从 `findSentenceForWord` 取，词在句中加粗高亮
- **通用例句**（灰色，**保留**）：原 `entry.example`

二者并存让学生既看到真实语境又看到一句变化的例句。无课文的词只显示通用例句。

实现：`src/components/english/words/FlashCard.tsx`。

### 1C. 闪卡课文模式

`/english/words/flash` 顶部新增 `[📖 课文模式]` 开关：

| 模式 | 正面 | 背面 |
|------|------|------|
| 标准（默认） | 词 + 释义 + 例句 | emoji + 中文 |
| 课文模式 | **例句挖空**（词用 `____` 遮住）+ 提示"语境填空" | （同标准背面） |

挖空使用现有的 `blankOutExample` 函数（含动词变形识别）。

实现：`src/app/english/words/flash/page.tsx#PassageModeContext`。

## 阶段二 — 阅读（During）

### 2A. 阅读页词高亮

四级掌握度着色：

| Level | 视觉 | 含义 |
|-------|------|------|
| 0 未掌握 | 橙底色 + 粗下划线 | 需要重点关注 |
| 1 学习中 | 蓝底色 + 下划线 | 正在记忆 |
| 2 熟悉 | 紫细下划线 | 已有印象 |
| 3 掌握 | 绿极细下划线 | 不干扰阅读 |

点击词弹出 `WordPopup`（移动端底部 sheet / 桌面居中弹卡）：词 + IPA + 释义 + 课文原句高亮 + 掌握度进度条。

### 2B. 段落末尾"遮词回想"小测验

每段读完后，**滚动超过段落 80%** 才出现回想题（用 `IntersectionObserver` + `rootMargin: '0px 0px 25% 0px'` 实现）：

```
💭 你还记得吗？
"Students _______ problems facing the plants..."

[discuss]  [solve]  [imagine]  [path]      ← 4 选 1
```

- **答对** → 绿色动效 + `recordBatch([{ entry, correct: true }])` 推进 masteryMap → 词色立即更新
- **答错** → 红色提示 + 展开原句（词高亮显示）+ **不修改** masteryMap
- 每段最多 1 次，挑该段中掌握度最低（level 0–1）的词
- 已答过的题不再重复

记忆原理：**主动提取（Active Recall）+ 间隔重复（Spaced Repetition）**，与练习共用同一 masteryMap。

### 2C. 两种阅读模式

阅读页顶部 toggle：

| 模式 | 行为 |
|------|------|
| 🧠 学习模式（默认） | 启用段落末尾回想题 |
| 📖 专注阅读 | 只显示词高亮，无回想题，纯阅读 |

切换通过 `renderParagraphFooter` prop 是否传入控制。

## 阶段三 — 复习（After）

### 3A. Type D 题型：课文语境填空

**现有 A/B/C：**

| 题型 | 展示 | 操作 |
|------|------|------|
| A | 中文释义 | 4 选 1 选词 |
| B | 英文词 | 4 选 1 选释义 |
| C | 中文释义 | 字母拼写 SpellTiles |

**新增 D：**

| 题型 | 展示 | 操作 | 差异 |
|------|------|------|------|
| D | **课文原句（挖空）** | 4 选 1 选词 | **有语境** |

```
📖 来自 Unit 5 · Lesson 2 课文

"Their _____________ isn't just about
 homework and exams. Students solve
 real world problems..."

[education]  [friendship]
[boarding school]  [understanding]

答对：✅ 正确！[📖 查看原文上下文 →] [下一题 →]
```

**触发条件**（与 ⭐ 无关）：该词的课次有课文 + 课文中能找到包含该词的句子。

实现位置：
- 题目生成：`buildQuizQuestions` 接受 `eligibleForTypeD` 谓词
- 独立练习入口：`ImmersiveMode.tsx`（深色全屏）
- 每日/每周练习入口：`QuizCard.tsx`（亮色卡片，被 WeeklyPlanSession 复用）

**星星奖励**：+2 红星/题（与 Type C 持平）。

### 3B. 任意题型答对都可"查看原文"（通用化）

凡是答完一道题、该词的课次有课文，**无论 A/B/C/D**，都在题卡底部显示：

```
[📖 查看课文原句 →]
[下一题 →]
```

- 答对 + 有课文 → 显示两个按钮（**禁用自动跳题**，让学生主动选择）
- 答对 + 无课文 → 现有自动跳题（600ms）保留
- 答错（任何） → "正确答案"提示 + 上述两按钮（若有课文）+ "下一题"

实现：`QuizCard.tsx` + `ImmersiveMode.tsx` 双处同步。

### 3C. 沉浸模式语境版入口

阅读页底部加 CTA：

```
🧠 开始语境练习  →
用本课课文原句挖空，检验你的语境理解
```

点击 → `/english/words/practice?context=<passageKey>`，practice 页检测到 query：
1. 自动筛选到该课次
2. 题型只勾选 Type D
3. 直接进入 ImmersiveMode 沉浸练习
4. 清掉 URL query（避免刷新时再次自动启动）

实现：`practice/page.tsx` URL 处理 + `reading/[key]/page.tsx` 底部 CTA。

## 计划模块

### 课程三标记 ⭐ / 必记 / 预习

选课行右侧并排三个按钮：

```
Unit 5 · Lesson 2    [⭐]  [必记]  [预习]
Unit 5 · Lesson 1         [必记]  [预习]
Unit 5 · Lesson 3         [必记]  [预习]
```

| 标记 | 含义 | 规则 |
|------|------|------|
| ⭐ 重点 | 本周核心课的标签 | 独立按钮，最多一课，只有必记课可选 |
| 必记 | 需要记忆 | 现有行为不变 |
| 预习 | 超前接触 | 现有行为不变 |

**与原方案的差异**：⭐ 不再决定任何模块的功能启用，仅作 UI 标签 + Today / 阅读 redirect 的软优先级。详见上方"⭐ 不闸门功能"章节。

### 数据层

```typescript
interface WeeklyPlan {
  // 现有字段
  previewLessonKeys?: string[]
  wordKinds?: Record<string, 'consolidate' | 'preview'>
  focusLessonKey?: string       // 新增，可选
}

interface WeekDayProgress {
  // 现有字段
  consolidateDone?: boolean
  previewDone?: boolean
  readingDone?: boolean          // 新增，scroll 50% 时写入
}
```

序列化已通过 `weeklyPlanPayload.ts` 完整支持。

## 三阶段完整数据流

```
课文索引（reading-data.ts）
│
├─── 阶段一（Before）
│     ├── 单词卡 → 课文原句 + 通用例句（共存）
│     ├── 闪卡  → 课文模式（例句挖空）
│     └── Today → 三步式 ①②③ 任务流
│
├─── 阶段二（During）
│     ├── 阅读页 → 词高亮（掌握度着色）
│     ├── 段落小测验 → 更新 masteryMap → 词色实时变化
│     └── 词弹卡 → 轻量查词
│
└─── 阶段三（After）
      ├── 任意题型 + 有课文 → "查看原文" 跳转
      ├── Type D 题 → 课文原句填空
      └── 阅读完 → 沉浸语境练习入口
```

**掌握度是唯一的状态**：阶段二的回想题答题、阶段三的练习答题，全部写入同一个 `masteryMap`。词的颜色在阅读页、单词卡、Today 页中实时反映最新状态。

## 验证步骤

1. **准备数据**：周计划必须有 U5L2（或任意 `reading-data.ts` 中存在的课次）作为今日新词。**⭐ 可标可不标**。
2. **阶段一**：
   - `/today` → 顶部出现 ①②③ 三行任务
   - `/english/words/cards` → 该课词卡同时显示课文原句 + 通用例句
   - `/english/words/flash` → 顶部点 `[📖 课文模式]` → 卡片正面变成例句挖空
3. **阶段二**：
   - `/english/words/reading/u5l2` → 词按掌握度高亮 → 顶部 toggle 切换专注/学习模式
   - 滚动到段落末尾 80% → 出现 💭 回想题 → 答对绿色 + 词色立即变化 → Today 卡片变绿（已读）
4. **阶段三**：
   - `/english/words/practice` → 任意 A/B/C 题答对 → 出现 "查看课文原句" + "下一题"
   - daily 进入练习 → "必记词题型" 行出现金色 D 按钮（默认勾选）→ 答对 D → 跳阅读页词闪烁
   - 阅读页底部 "🧠 开始语境练习" → 自动进入 Type D 沉浸全屏
5. **跨阶段闭环验证**：在阅读页回想题答对一次 + 在练习答对一次 → masteryMap 累加 2 → 单词卡的掌握度图标更新

## 设计原则总结

**这个方案的关键在于：不是新建三个孤立功能，而是用课文句子索引把已有模块串联起来。**

- 掌握度系统（masteryMap + spaced repetition）不变
- 新增一个数据字段 `focusLessonKey` 作为计划标签
- 新增一个数据字段 `readingDone` 跟踪每日"先读课文"完成
- 新增一个 `ReadingPassage` 索引文件 + 一个 `WeekDayProgress` 字段
- 其他全部是 UI 层在已有数据上做更聪明的展示

添加新课文是**纯数据改动**：往 `readingPassages` 数组追加一项，所有功能（高亮、Type D、单词卡原句、闪卡课文模式、Today 推荐、查看原文跳转）自动接上。
