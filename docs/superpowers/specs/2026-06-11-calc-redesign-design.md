ok# 口算系统重构 · 设计文档

**日期:** 2026-06-11
**模块:** `/calc`(口算)
**状态:** 已与用户确认,待细化实施计划

---

## 1. 背景与目标

现有口算系统把"运算 + 数字范围"捆死在 Lv.1–20 的线性阶梯里,并叠加了一套重型机制:
题库 bank 种子池、掌握度间隔复习(R1/R2/R3)、关卡 A/B/C 状态机、自适应升降档、攻坚模式、事件日志。

用户要求把出题从"线性阶梯"改成**两个可组合的维度**,并大幅简化:

1. **去掉"运算类型"选择器**——运算类型直接由"挑选题型"决定。
2. **难度顺序应体现运算固有难易**:加 < 减 < 乘 < 除 < 混合。
3. **混合运算用"编排器"自定义**:从单运算积木块拼出混合题,可加多条。
4. **保留薄弱点感知**:出题时夯实薄弱计算、熟练的少出,练完出报告看各部分掌握度,反哺下次出题。
5. **题量选项进入设置页**。

## 2. 范围

**做:** 出题模型重构、积木库、单运算多选、混合编排器(7 骨架)、轻量薄弱点追踪与加权、报告改造、题量入设置、设置页重排、数据迁移、删除/停用旧机制。

**不做(保留现状):** 错题本(`/calc/mistakes`)、星星/金币钱包、限时奖励、答题音效、PWA、AST 出题底层工具。

## 3. 核心模型

出题 = **积木块**(纯单运算,自带难度)+ **混合编排**(7 个固定骨架,从积木拼出)。

### 3.1 积木库(building blocks)

每个积木块 = 一个**纯单运算** + 一个**难度范围**,带 `op` 标签(供混合编排筛选)。难度天然满足 加 < 减 < 乘 < 除 的顺序。

| 运算 | 积木块(由易到难) | 块 ID |
|---|---|---|
| **加法** `op=add` | 10 内 / 20 内不进位 / 20 内进位 / 100 内不进位 / 100 内进位 | `add:10` `add:20a` `add:20b` `add:100a` `add:100b` |
| **减法** `op=sub` | 10 内 / 20 内不退位 / 20 内退位 / 100 内不退位 / 100 内退位 | `sub:10` `sub:20a` `sub:20b` `sub:100a` `sub:100b` |
| **乘法** `op=mul` | ×2,5 / ×3,4 / ×6,7 / ×8,9 / 2-9 综合 / ×10-12 / ×13-19 / 2-19 综合 | `mul:25` `mul:34` `mul:67` `mul:89` `mul:29` `mul:1012` `mul:1319` `mul:219` |
| **除法** `op=div` | ÷2,5 / ÷3,4 / ÷6-9 / ÷2-9 综合 / ÷10-12 / ÷13-19 / ÷2-19 综合 | `div:25` `div:34` `div:69` `div:29` `div:1012` `div:1319` `div:219` |

- "加减法 / 乘除法"只是按 `op` 标签聚合出来的**分类视图**,不是独立实体。
- 每个块导出两个能力:
  - `generateSingle(): CalcQuestion` —— 单运算练习直接用。
  - `sampleTerm(): { ast, value }` —— 供混合编排取一个该运算的子项(乘块给 `b×c`,除块给 `dividend÷divisor`,加/减块给一个落在该范围的操作数 + 允许的运算符)。
- 块的难度范围逻辑复用现有 `lesson`/`calc-levels` 里的生成代码(`genL1..genL13/16/18/19` 的数字范围约束),按 op 拆分重组(现有 addsub 档一题随机出 +/−,需拆成纯加块和纯减块)。

### 3.2 单运算练习

设置页里加 / 减 / 乘 / 除四组,**直接多选积木块**,不走编排。选中的块混着出题。

### 3.3 混合运算 = 编排器(7 固定骨架)

括号是**骨架的一部分**,不是开关。固定 7 个骨架,对齐课程进阶:

| # | 骨架 | 形状 | 编排时要选的积木维度 |
|---|---|---|---|
| 1 | 加减混合(同级) | `5 + 3 − 2` | 加块 / 减块 |
| 2 | 乘除混合(同级) | `6 ÷ 2 × 3` | 乘块 / 除块 |
| 3 | 加减与乘法(两级) | `3 + 4 × 2` | 加减块 + 乘块 |
| 4 | 加减乘除全混合(两级) | `3 + 4 × 2 − 6 ÷ 3` | 加减块 + 乘块 + 除块 |
| 5 | 加减与乘法·带括号 | `(3 + 4) × 2` | 加减块 + 乘块 |
| 6 | 乘除·带括号 | `(8 ÷ 4) × 2` | 乘块 + 除块 |
| 7 | 加减乘除·带括号 | `(2 + 4) × 3 ÷ 2` | 加减块 + 乘块 + 除块 |

**编排流程:**

1. 点"添加混合运算" → 选 1 个骨架。
2. 系统按骨架声明的"要选的积木维度",用 `op` 标签从积木库**筛出可用块**展示。
3. 每个涉及维度**至少多选 1 块**(否则该条无效)。
4. 存为一条混合运算(自动命名,如「加减乘·括号 · 20内 + 6,7乘」),可**启用/停用/删除**。可加多条。

**出题映射:** 每个骨架有"加减位 / 乘除位"槽位。选中的积木块决定:
- 各槽位**允许哪些运算符**(选了减块 → 加减位可出 −;选了乘块 → 乘除位可出 ×)。
- 各槽位**操作数多大 / 用哪套口诀**(由块的范围决定)。

约束:生成时保证无负中间值、整除(除法槽)、结果非负整数;失败则重采样(沿用现有 `genTwoOp` 的重试思路)。

### 3.4 难度顺序的体现

不再有独立的 ★ 难度轴。难度由"选哪些积木 + 选哪些骨架"决定;加 < 减 < 乘 < 除 < 混合 的固有顺序通过**积木库分组顺序**与**骨架排序**在 UI / 报告里体现。

## 4. 薄弱点追踪与反哺(保留的轻量层)

**复用 `calc_problem_state` 表**,**砍掉** R1/R2/R3 间隔复习、A/B/C 状态机、自适应升降档。

- **记录:** 每题作答后更新该 `signature` 的熟练度(对错 + 速度,0–5)。给状态行标注它属于哪个**积木块 ID**(单运算)或哪条**混合配置 ID**。
- **单运算块加权:**
  - 题量在选中的积木块之间按 `权重 = 1 − 块平均熟练度` 分配(弱块多出),每个选中块**保底 ≥ 1**。
  - 块内约 **30–40%** 名额优先**复现**该块下近期做错 / 低熟练度的具体 `signature`,其余现场随机新生成;已熟练 signature 不再复现。
- **混合条加权:** 混合算式近乎无穷,**按"每条混合配置"聚合熟练度**做条间加权与报告展示;题面仍随机新生成(不做 signature 级复现)。

## 5. 报告

报告分两层:**练完即时报告**(每次 session 结束)与**总览报告**(`/calc/report`,随时查看)。
两层**共用同一份 per-signature 熟练度聚合**,且与第 4 节出题加权**同源**——报告里看到的"弱",正是下次会被多出的题。这是用户"看掌握度 → 反哺出题"诉求的落点。

### 5.1 熟练度口径(报告与出题加权共用)

- per-signature:熟练度 0–5 + status(未练 / 练习中 / 已熟练),来自 `calc_problem_state`。
- **积木块熟练度**:
  - **乘 / 除块(口诀有限)** → 覆盖率口径 = 已熟练口诀数 / 该块全部口诀数(如 `mul:67` 共 16 条:6×2..6×9、7×2..7×9)。
  - **加 / 减块、混合条(空间大 / 近乎无穷)** → 平均熟练度口径 = 已作答 signature 的熟练度均值(未作答不计入)。
- **混合条熟练度**:按 `mixed_op_id` 聚合,平均熟练度口径。

### 5.2 练完即时报告(`SessionSummary` 扩展)

在现有概况(总题数 / 首答对率 / 用时 / 平均每题时间 avgMs vs prevAvgMs / 连击 / 星星)之上,新增:

- **本次分解(按出题源)**:每个参与的积木块 / 混合条一行——出题数、首答对数、**熟练度变化(↑ / ↓ / →)**。
- **本次新暴露薄弱**:本次做错的具体算式列表(最多约 8 条)。
- **下次预告**:基于更新后的熟练度,列出下次将被**加权多出**的最弱 3–5 项(块或混合条),直接呼应"反哺"。

### 5.3 总览报告(`/calc/report` 改造)

去掉"关卡状态(Lv.1–20)"与"关键事件时间线"(无 level 状态机 / 事件日志了),改为:

1. **掌握度总览**:按 `加 / 减 / 乘 / 除 / 混合` 分组。
   - 单运算:每个**积木块**一行——熟练度 %(按 5.1 口径)、作答数、状态条;点击**展开**看该块下最弱的具体算式(复用现有展开式 UI)。
   - 混合:每条**混合配置**一行——聚合熟练度 %、作答数;展开看该条最近做错的算式。
2. **最弱 N 题**(默认 10):跨所有块 / 混合的最弱具体算式(复用现有 weakest 逻辑:proficiency 升序 + 连错降序)。
3. **最近练习**:复用 `wallet.sessions`,列最近若干次 session 的日期 / 题数 / 对数 / 星星(替代原"关键事件")。
4. *(可选,锦上添花)* **趋势**:最近 N 次 session 的首答对率 / 平均每题用时折线。

### 5.4 数据来源

- `calc_problem_state`(signature、proficiency、status、`block_id`、`mixed_op_id`、consecutive_wrong)——总览主数据。
- `calc_sessions`——最近练习 / 趋势。
- 错题本(`calc_mistakes`)维持独立入口,不并入报告。

## 6. 题量 / 限时 / 音效进设置页

设置页内置 `CalcConfigBar`(题量 10/20/30/50/100 + 限时),持久化到现有 `lastCount` / `lastTimeLimit`。首页 `/calc` 的 `CalcConfigBar` 保留。

## 7. 数据类型与 Schema

### 7.1 `CalcSettings` 变更

**删除:** `enableAddSub` `enableMulDiv` `enableMixed` `currentLevel` `adaptive` `freeMode` `freeModeLevels`。

**新增:**
```ts
selectedBlocks: string[]        // 单运算练习选中的积木块 ID,如 ['add:20b','mul:67']

```

**保留:** `soundEnabled` `lastCount` `lastTimeLimit` `timeLimitOverrides` `sessionCounter`。

### 7.2 新类型

```ts
export type CalcSkeletonId =
  | 'as'          // 加减混合
  | 'md'          // 乘除混合
  | 'asm'         // 加减与乘法
  | 'asmd'        // 加减乘除全混合
  | 'as_m_paren'  // 加减与乘法·带括号
  | 'md_paren'    // 乘除·带括号
  | 'asmd_paren'  // 加减乘除·带括号

export interface MixedOp {
  id: string                 // uuid,稳定标识(用于薄弱度聚合)
  skeleton: CalcSkeletonId
  blockIds: string[]         // 选中的积木块 ID(跨维度)
  enabled: boolean
  label?: string             // 可选自定义名;缺省自动生成
}
```

`CalcCategory` / `CalcLevel` 等旧类型按需保留或收敛(见删除清单)。

- `calc_problem_state`:加列 `block_id text`、`mixed_op_id text`(标注该 signature 归属);`level` 列保留兼容。
- `calc_level_state` / `calc_event_log`:**停止写入,不 drop**(非破坏)。

## 9. 出题引擎

### 9.1 出题策略(完整算法)

**输入:** 选中积木块、启用且有效的混合条、题量 `count`、每个 signature 的熟练度(含 `block_id` / `mixed_op_id` 归属)、未解决错题(`calc_mistakes`)。

**⓪ 跨次错题补练(决定本场总题量):**
- 取**上一次 session 遗留的未解决错题**(`calc_mistakes` 未 resolved,归属最近一次完成的 session;用 `lastWrongAt` 或在 mistake 上记 `sessionNo` 识别)。记数 `M`。
- **本场总题量 = 配置 count + M**(错题**叠加**在 count 之上,不挤占正常生成)。这 M 道以**原错题面**作为补练段纳入本场。
- 答对沿用现有逻辑:连续 3 次对 → 移出错题本(resolved)。本场再次答错 → 进末尾补做(⑤),并作为"本次错题"留给**下一场**继续叠加,直到 resolved。
- 安全上限(可选):`M > count` 时截断为 count,避免单场过长。无遗留错题则 `M = 0`。
- 与策略③分工:**错题 = 明确做错过的硬清单(跨次保证补练)**;复现薄弱 = 低熟练度软信号。

**① 出题源(sources):** 每个选中积木块 = 一源;每条启用混合条 = 一源。全空 → 兜底 `add:10`。正常生成用完整 `count` 在源间分配(与 ⓪ 的 M 错题段叠加,本场共 `count + M` 题)。

**② 题量在源间分配(弱者多出 + 保底):**
- 每源聚合熟练度 `p`(报告 5.1 口径),权重 `w = 1 − p/5`(从未练过 → `w = 1`)。
- `count ≥ 源数`:每源保底 1 题,余量按 `w` 比例分(最大余数法)。
- `count < 源数`:取最弱的 `count` 个源各 1 题。

**③ 源内生成:**
- **单运算块**(N 题):约 **35%** 名额复现该块下薄弱具体算式(proficiency 升序 + 连错降序 + 近期错优先,排除已熟练;不足补新生成);其余 block.`generateSingle()` 现场随机新生成,避免本场重复。
- **混合条**(N 题):全部经骨架组装器现场新生成;薄弱反哺由 ② 的"弱条多出"承担。

**④ 非负 / 整除约束(混合):** 组装时调整运算符方向与操作数顺序,保证结果非负整数、除法整除;采样失败重采。
> 例:混合条「减:20内」+「乘:6,7」生成 `a − 6×7`,若 `a≤20` 会负 → 改 `6×7 − a`,仍属"加减与乘法"骨架。

**⑤ 去重 / 打乱 / 错题追加到末尾:** 收齐后 Fisher-Yates 打乱(错题补练题混入,不固定排最前)。**运行时答错的题不再 +4 插队,而是在本场计划题全部做完后,统一追加到末尾重做**(再次答错则再次追加,直至答对)——替代旧 `interleaveWrong` 的就地插队。单运算沿用 `pickForm`/`applyForm` 题面变体,混合用标准形。
> 进度条 / 计数:计划题为分母;追加的错题补做段作为"加时"段单独提示,不计入原 `count` 进度,但计入 session 实际作答统计。

**⑥ 熟练度更新(喂回 ②③):** 复用现有 `applyAttempt` 的 0–5 熟练度部分,**砍掉 R1/R2/R3 复习排程**。首答对且限时内 → +1(上限 5);对但慢/重试 → 微增;答错 → −1~2(下限 0)、连错 +1。status:充分作答且 proficiency ≥ 4 → 已熟练;0 次 → 未练;其余 → 练习中。

## 11. 默认值与边界

- **空选兜底:** 未选任何积木块且无启用混合条 → 退回 `add:10` 单块,避免空 session。
- **混合条不完整:** 某维度未选块 → 该条标记无效、出题跳过,设置页提示。
- **块拆分:** 现有 addsub 生成器一题随机 +/−,拆分时加块只出 +、减块只出 −,数字范围沿用对应档。
- **金币:** 复用现有 `coinBase`——单运算块按运算类别给基础币(加减 1、乘除 2),混合条按骨架复杂度给(同级 2、两级 3、带括号 4),叠加连击加成。挑战题 ×2 概念并入带括号骨架。

## 12. 测试 / 验证

无自动化测试套件。验收要点:
- 每个积木块单独出题数字范围正确、纯单运算。
- 7 骨架各自结构正确、括号/运算顺序正确、无负中间值、除法整除。
- 薄弱加权:弱块明显多出、熟练块少出;块内复现近期错题。
- 错题:本场答错追加到末尾重做(直至答对);**下一场总题量 = 配置 count + 上场未解决错题数**,这些错题随之补练;连对 3 次 resolved 后不再带入。
- 报告:练完即时 + 总览均按新维度展示。
- 迁移后老用户配置不崩(缺新列时走默认)。
- `pnpm lint` 通过;UI 改动后 `pnpm build` 通过。
