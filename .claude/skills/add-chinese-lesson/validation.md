# 语文课次内容校验（网络检索）

本文件是 `/add-chinese-lesson` 的**强制关卡**。Agent 必须在写入 TS / Python 数据前完成校验，并在对话中输出校验报告。

## 原则

1. **字词句不能有错别字** — 每个汉字与权威源一致
2. **拼音不能错** — 声调正确；多音字用**该课**教材读音
3. **古诗/韵文不能错行** — `lines[]` 或 passage 分行与教材一致（该合并的合并，该分行的分行）
4. **读一读记一记** — 整句一致，不可拆字、不可漏字
5. **至少两个独立来源交叉验证**；单来源不足以通过

## 工具

- **WebSearch** — 发现权威页面 / PDF 链接
- **WebFetch** — 读取页面正文对照（优先可复制的纯文本段）

不要使用「模型记忆」代替检索。部编版课文偶有修订，以检索到的现行教材为准。

## 推荐检索词（按内容类型）

将 `{G}` `{S}` `{U}` `{标题}` 替换为实际值（如 二、上、3、树和喜鹊）：

| 类型 | 检索词示例 |
|------|-----------|
| 课文全文 | `部编版 {G}年级{S}册 {标题} 课文 原文` |
| 课文全文（备用） | `人教版 语文 {G}年级{S} {标题} 课文` |
| 识字表 | `部编版 {G}年级{S}册 {标题} 识字表 会认` |
| 写字表 | `部编版 {G}年级{S}册 {标题} 写字表 会写` |
| 古诗 | `{诗名} {作者} 原文` + `部编版 {G}年级{S}册 {诗名}` |
| 读一读记一记 | `部编版 {G}年级{S}册 {标题} 读一读 记一记` |
| 语文园地 | `部编版 {G}年级{S}册 语文园地{U}` |

优先来源（降序）：
1. 人民教育出版社 / 国家中小学智慧教育平台
2. 标注「部编版」「统编版」的教材 PDF 或教师用书摘录
3. 大型教辅站（如教习网、21 世纪教育网）的课文原文页 — 仅作交叉验证，单站不可独占

避免仅引用：贴吧、个人博客、无出处的 OCR 乱码页。

## 校验流程（每课必做）

```
Task Progress:
- [ ] 1. 列出本课待校验 payload（见下方清单）
- [ ] 2. 检索课文原文 ≥2 源
- [ ] 3. 检索识字/写字表 ≥2 源（或 PDF 目录页）
- [ ] 4. 若有古诗：检索古诗原文 + 分行
- [ ] 5. 若有 recall_phrases：逐句检索或在与课文同源 PDF 中核对
- [ ] 6. 填校验报告；处理 🔴 项
- [ ] 7. 全部 🟢 后才可写入数据文件
```

### 待校验 payload 清单

从用户源提取后，在写入前逐项列出：

```markdown
## 待校验：u3-l5 《树和喜鹊》

### 课文（N 段）
1. ...
2. ...

### 认读字（N）
霜 吹 落 ... + 拼音

### 会写字（M）
...

### 读一读记一记
- 叽叽喳喳
- 安安静静

### 古诗（如有）
《xxx》作者 — 共 K 行
```

## 对照规则

### 汉字

- 逐字比对；注意形近字（已/己、未/末、人/入）
- 繁体源必须转为简体后再比
- 标点：教材用 `，` 不用 `,`；引号用 `「」` 或 `""` 与教材一致
- 省略号：教材 `……`（六个点）vs `...` — 跟教材

### 拼音

- 带声调：`xī` 不是 `xi`
- ü 写作 `u` 或 `ü` 与项目现有数据一致（如 `g1b` 字库用 `xu`、`nǚ` 等标准形式）
- 多音字：以**该课生字表标注**为准，不取通用首选读音
- `lesson-chars` 内拼音顺序与 `recognize`/`write` 数组严格一一对应

### 古诗 / 韵文分行

- 每行一条 `lines[]` 或 passage 中的一段
- 与教材断句一致（如「床前明月光，」一行 vs 分成两行 — 跟教材）
- 诗名、作者、朝代与册内定位一致

### 课文段落

- `curated_passages_data.py` 中每个 string = UI 一个 `<p>`
- 对话体可以一行一段，也可以按教材自然段合并 — **以教材段落为准**
- 校验时把网络源的段落切分方式与拟写入的 `paragraphs[]` 对齐

## 校验报告模板（必须在对话中输出）

```markdown
# 内容校验报告 — u3-l5 《树和喜鹊》

## 检索来源
1. [来源名称](URL) — 用途：课文原文
2. [来源名称](URL) — 用途：识字写字表

## 对照结果

| 项目 | 状态 | 说明 |
|------|------|------|
| 课文全文 | 🟢 | 2 源一致，共 6 段 |
| 认读字 12 个 | 🟢 | 与识字表一致 |
| 会写字 8 个 | 🟢 | 与写字表一致 |
| 拼音 | 🟢 | 多音字「乐」本课读 lè，已确认 |
| 读一读记一记 | 🟢 | 2 句一致 |
| 古诗 | — | 本课无 |

## 差异与处理
（无则写「无」）

## 结论
🟢 通过，可写入 TS 备份
```

状态含义：
- 🟢 通过 — 多源一致或用户已确认采用某一源
- 🟡 待确认 — 源之间有差异，已列出选项等用户决定
- 🔴 不通过 — 无法找到可靠来源、或与用户要求冲突 — **禁止写入**

## 常见 🔴 场景

| 场景 | 处理 |
|------|------|
| 网络源 A/B 汉字不同 | 列差异，查第三源；仍无法确定 → 问用户 |
| 用户 OCR 缺字 | 以网络教材补全，报告注明 |
| 修订版课文（老教辅 vs 新教材） | 优先新教材 / 智慧教育平台；注明版本 |
| 仅找到图片 PDF 无法 fetch | 换检索词找 HTML 原文页，或请用户提供 PDF |
| 生字表网页缺字 | 换检索「写字表」「识字表」PDF |

## 校验通过后的最小留痕

在本课相关的 `curated_passages_data.py` 或 `docs/chinese/.../lessons/<key>.md` 顶部加一行注释：

```python
# u3-l5 — 校验 2026-07-02：课文/生字与部编版二上一致（来源：xxx, xxx）
```

便于日后审计，非必须但推荐。
